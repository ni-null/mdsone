# 快速開始

## 安裝

```bash
# 透過 npx 執行（推薦）
npx mdsone [輸入] [選項]

# 或全域安裝
npm install -g mdsone
```

## 常見用法

### 單一 Markdown 檔案

```bash
# 輸出到目前目錄（例如 README.html）
npx mdsone README.md

# 指定輸出檔案
npx mdsone README.md -o index.html
```

### 多個 Markdown 檔案

```bash
# 批次模式：每個 .md 各自輸出 .html
npx mdsone a.md b.md c.md

# 輸出到指定目錄
npx mdsone a.md b.md -o ./dist
```

### 資料夾

```bash
# 批次模式：資料夾中每個 .md 各自輸出（-o 必須是目錄）
npx mdsone ./docs -o ./dist

# 合併模式：整個資料夾輸出成單一 HTML
npx mdsone ./docs -m
```

### 合併模式

```bash
# 多個檔案合併
npx mdsone intro.md guide.md -m

# 合併並指定輸出檔名
npx mdsone intro.md guide.md -m -o manual.html

# 資料夾合併並指定模板變體
npx mdsone ./docs -m -o manual.html --template normal@warm-cream
```

### 常用功能

```bash
# 啟用圖片 base64 嵌入
npx mdsone ./docs -m --img-embed=base64

# 顯示程式碼行號
npx mdsone ./docs -m --code-line-number

# 使用區塊複製模式
npx mdsone ./docs -m --code-copy=cmd

# 若輸出檔已存在則停止
npx mdsone README.md -o output.html

# 強制覆蓋既有輸出
npx mdsone README.md -o output.html --force
```

## 兩種運作模式

| 模式 | 說明 |
|------|------|
| 批次模式（預設） | 每個 Markdown 檔案各自產生獨立的 HTML |
| 合併模式（`-m`） | 所有輸入合併為單一 HTML，以分頁呈現 |

## 設定優先順序

```text
CLI 參數 > 環境變數 > config.toml > 預設值
```

> 環境變數只讀取 `process.env`（shell / Docker / CI 注入），不會自動載入 `.env` 或 `.env.local`。

## 目前內建模板

目前內建模板為 `normal`，並可透過 `--template` 搭配 `@variant` 切換模板變體，例如：

```bash
npx mdsone ./docs -m --template normal@default
npx mdsone ./docs -m --template normal@fog-gray
npx mdsone ./docs -m --template normal@warm-cream
```
