# Templates

## 內建模板

| 名稱 | 說明 |
|------|------|
| `normal` | 側邊欄 + TOC + 亮暗色切換 |
| `minimal` | 簡潔排版，適合輕量文件 |

```bash
npx mdsone --template normal
npx mdsone --template minimal
```

## 代碼高亮主題

代碼高亮在 **build 時完成**（伺服器端），而非在瀏覽器 runtime 執行。這意味著：

- ✅ 無需在輸出 HTML 中嵌入 highlight.js（節省 ~120KB Javascript）
- ✅ 無 runtime 錯誤風險
- ✅ 更快的頁面加載速度

### 可用主題

highlight.js 提供 **100+ 個官方主題**，來自 npm 套件。預設 dark/light 主題配對為：
- **Dark**（預設）：`atom-one-dark`
- **Light**（預設）：`atom-one-light`

### 自訂主題

透過 CLI 參數或 config.toml 指定任意主題名稱：

**CLI 方式：**
```bash
# 使用 GitHub Dark 和 GitHub Light 主題
npx mdsone --code-highlight-theme github-dark --code-highlight-theme-light github

# 或只更改深色主題
npx mdsone --code-highlight-theme nord
```

**config.toml 方式：**
```toml
[build]
code_highlight_theme       = "github-dark"
code_highlight_theme_light = "github"
```

## 模板結構

```
templates/
└── my-template/
    ├── template.html
    ├── style.css
    ├── template.config.json
    └── assets/
        ├── example1.css      # 自動掃描並注入
        └── example2.js       # 依數字前綴排序
```

### assets/ 資料夾

`assets/` 內的 CSS/JS 檔案會自動掃描並 inline 注入到模板中，無需在 `template.config.json` 明確列舉：

- **CSS 檔案** → 注入為 `<style>` 在 `<head>` 內
- **JS 檔案** → 注入為 `<script>` 在 `</body>` 前
- **排序規則** → 檔名依數字前綴排序（例：`01_base.css`、`02_theme.css`）

這是處理模板專屬樣式和邏輯的最簡單方式。

## template.config.json

```json
{
  "_metadata": {
    "version": "1.0.0",
    "schema_version": "v1"
  },
  "toc": { 
    "enabled": true, 
    "levels": [2, 3] 
  }
}
```

| 欄位 | 說明 |
|------|------|
| `_metadata.version` | 模板版本（資訊用） |
| `_metadata.schema_version` | 模板配置格式版本 |
| `toc.enabled` | 是否顯示目錄 |
| `toc.levels` | 目錄涵蓋層級，如 `[2, 3]` 代表 h2、h3 |

> 額外的 CSS/JS 可以放在 `assets/` 資料夾中自動掃描，或透過 CLI / 環境變數 / config.toml 中的 `--extra-css` / `--extra-js` 選項指定外部 URL。

## 新增自訂模板

```bash
# Windows PowerShell
Copy-Item -Recurse templates/minimal templates/my-template

# macOS / Linux
cp -r templates/minimal templates/my-template

# 編輯模板檔案
cd templates/my-template
# 編輯 template.html、style.css、locales 等

# 執行自訂模板
npx mdsone --template my-template

# 或在 config.toml 中指定
# [build]
# default_template = "my-template"
```

## template.html 佔位符

| 佔位符 | 替換內容 |
|--------|----------|
| `{TITLE}` | 頁面標題 |
| `{LANG}` | HTML lang 屬性 |
| `{CSS_CONTENT}` | style.css 內容 |
| `{LIB_CSS}` | lib/ 資料夾中的樣式（highlight.js 主題） |
| `{EXTRA_CSS}` | 額外 CSS 標籤 |
| `{LIB_JS}` | lib/ 資料夾中的腳本（highlight.js 和複製按鈕） |
| `{EXTRA_JS}` | 額外 JS 標籤 |
| `{MDSONE_DATA_SCRIPT}` | 文件資料 JSON（window.mdsone_DATA 事件） |

### lib/ 資料夾

`lib/` 用來存放可選功能的程式碼檔案：

```
lib/
├── highlight/
│   ├── highlight.min.js       # highlight.js 語法高亮引擎
│   └── css/
│       ├── atom-one-dark.min.css    # 深色主題
│       └── atom-one-light.min.css   # 淺色主題
└── copy/
    └── copy.js                # 程式碼複製按鈕功能
```

根據 `--code-highlight` 和 `--code-copy` 旗標，對應的檔案會被 inline 注入到 `{LIB_CSS}` 和 `{LIB_JS}` 中。用戶可透過參數動態啟用/禁用這些功能。
