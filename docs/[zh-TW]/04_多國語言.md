# i18n

## 內建語系

| 代碼 | 語言 |
|------|------|
| `en` | English |
| `zh-TW` | 繁體中文 |

```bash
npx mdsone --locale zh-TW
```

## 語系檔案結構

```text
locales/
├── en.json
└── zh-TW.json
```

```json
{
	"cli": {
		"err_md_dir_missing": "找不到來源資料夾：{dir}"
	},
	"template": {
		"html_lang": "zh-TW",
		"footer_label": "建置日期：{BUILD_DATE}"
	}
}
```

| 區塊 | 用途 |
|------|------|
| `cli` | 工具 log 與錯誤訊息 |
| `template` | 嵌入 HTML 的 UI 文字 |

## 新增語系

```bash
# Windows PowerShell
Copy-Item locales/en.json locales/ja.json

# macOS / Linux
cp locales/en.json locales/ja.json

# 編輯後執行
npx mdsone --locale ja
```

找不到指定語系時會自動 fallback 到 `en`。

## 多語言模式（I18N_MODE）

### 啟用方式

在 `config.toml` 設定（推薦）：

```toml
[i18n]
mode = true
default_locale = "zh-TW"
```

或透過環境變數：

```env
I18N_MODE=true
DEFAULT_LOCALE=zh-TW
```

### 來源目錄結構

來源資料夾下改用 `[locale]` 命名的子目錄分放各語系文件：

```text
docs/
├── [zh-TW]/
│   ├── 01_intro.md
│   └── 02_usage.md
└── [en]/
    ├── 01_intro.md
    └── 02_usage.md
```

> 目錄名稱必須使用方括號 `[locale]` 格式，例如 `[zh-TW]`、`[en]`、`[ja]`。

### 執行

```bash
npx mdsone --source ./docs
```

工具會自動掃描所有 `[locale]` 子目錄並將各語系文件打包進同一個 HTML。

### 輸出行為

- 生成的 HTML 右上角內建語言切換下拉選單（`<select>`）
- 切換語系時頁面即時更新，不需重新載入
- 使用者的語系選擇會儲存在 `localStorage`，下次開啟時自動套用
- `default_locale`（config.toml）或 `DEFAULT_LOCALE`（env）指定首次開啟時的預設顯示語系

### 單語 vs 多語模式資料結構

| 模式 | `window.mdsone_DATA` 結構 |
|------|--------------------------|
| 單語（預設） | `{ docs: [...], config: {...}, i18n: {...} }` |
| 多語 | `{ locales: [...], defaultLocale: "...", docs: { "zh-TW": [...], "en": [...] }, config: {...}, i18n: { "zh-TW": {...}, "en": {...} } }` |
