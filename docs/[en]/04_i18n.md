# i18n

## Overview

i18n mode bundles multiple locale document sets into one HTML and provides a built-in language switcher.

## Folder Layout

Use `[locale]` folder names under the source directory:

```text
docs/
  [en]/
    01_quickstart.md
  [zh-TW]/
    01_intro.md
```

## Enable i18n Mode

```bash
# Enable and set default locale
npx mdsone ./docs --i18n-mode=zh-TW -o dist/index.html
```

Notes:

- `--i18n-mode` enables i18n mode.
- `--i18n-mode=CODE` also sets default locale.
- In i18n mode, merge behavior is used automatically.

## Locale Sources

### Global locale names

`locales/config.json` maps locale code -> display name for UI.

Example:

```json
{
  "en": "English",
  "zh_TW": "Traditional Chinese"
}
```

### CLI + shared locale strings

`locales/<code>.json` provides CLI and shared i18n strings.

### Template locale strings

`templates/<name>/locales/<code>.json` provides template UI strings.

## Runtime Behavior

Generated HTML includes:

- locale list
- per-locale document payload
- per-locale i18n strings
- language switcher UI

Language selection is persisted in `localStorage`.

## config.toml

```toml
[i18n]
mode = true
default_locale = "zh-TW"
locale = "zh-TW"
```

## Environment Variables

```env
I18N_MODE=true
DEFAULT_LOCALE=zh-TW
LOCALE=zh-TW
```

Environment values are read from runtime `process.env` only (shell/Docker/CI). `.env` files are not auto-loaded.
